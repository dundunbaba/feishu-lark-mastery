# 通用知识：限频规则汇总

**来源**：[官方] open.feishu.cn
**最后更新**：2026-03-25

---

## 各模块限频

| 模块 | 接口 | 限频 | 备注 |
|------|------|------|------|
| 消息 | 发送消息 | 5 QPS/单聊，5 QPS/群（群内机器人共享） | |
| 消息 | 全局 | 1000次/分，50次/秒 | |
| 多维表格 | 记录 CRUD | 50 次/秒 | |
| 多维表格 | 同一张表写入 | 串行（不支持并发） | 并发写会报 1254291 |
| 云文档 | 获取所有块 | 5 QPS/应用 | |
| 云文档 | 创建/更新块 | 5 QPS/应用 | |

## 限频应对策略

1. **批量操作**：能用批量接口就不用单条（如 batch_create 代替循环 create）
2. **串行写入**：同一张多维表格的写操作必须串行，不能并发
3. **指数退避**：被限频后，等待 1s → 2s → 4s → 8s 重试
4. **错误码识别**：
   - `99991400` = HTTP 频率限制
   - `1254290` = 多维表格请求过快
   - `230020` = 消息频率限制
