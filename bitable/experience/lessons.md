### [2026-03-25] 字段名必须精确匹配
- **场景**：创建记录时传入字段名
- **问题**：报错 1254045（字段名不存在）
- **原因**：字段名多了空格或特殊字符，肉眼看不出差异
- **解决**：先调用 `feishu_bitable_list_fields` 获取精确字段名
- **教训**：**每次写记录前必须先 list_fields**

### [2026-03-25] 日期字段必须用毫秒时间戳
- **场景**：写入日期字段
- **问题**：传入秒级时间戳，日期显示异常
- **原因**：飞书多维表格日期字段统一要求**毫秒级**时间戳
- **解决**：时间戳 × 1000
- **教训**：日期字段永远用毫秒时间戳

### [2026-03-25] 人员字段格式要求数组
- **场景**：写入人员字段
- **问题**：传字符串 "ou_xxx" 报错 1254066
- **原因**：人员字段必须是 `[{"id": "ou_xxx"}]` 格式
- **解决**：改为数组对象格式
- **教训**：人员字段 = 数组 + 对象 + id 键

### [2026-03-25] 同一张表不能并发写入
- **场景**：批量写入记录
- **问题**：报错 1254291（Write conflict）
- **原因**：多维表格底层串行处理，不支持并发写入
- **解决**：改为顺序执行
- **教训**：写操作必须串行，包括新增/修改/删除记录和字段

### [2026-03-27 15:35] 多维表格内无法通过 API 创建文档
- **场景**：用户要求在多维表格左侧菜单创建「文档」Sheet
- **问题**：尝试了 feishu_bitable_create_app、feishu_doc 等工具，都无法在多维表格内创建文档
- **原因**：飞书多维表格的「文档」Sheet 是界面级功能，不是 API 能力。多维表格和云文档是两种独立的产品形态。
- **解决方案**：
  1. **方案 A（推荐）**：创建独立云文档，在多维表格描述中添加链接
  2. **方案 B**：在多维表格中添加「说明」表（用数据表形式存储规则）
  3. **方案 C（用户手动）**：用户在飞书界面点击「新建」→「文档」（但这创建的是独立云文档，不是表格内的）
- **教训**：
  - ✅ 多维表格 ≠ 云文档，两者是独立产品
  - ✅ 多维表格的「文档」Sheet 实际是「关联的云文档」，需要手动创建
  - ✅ API 只能创建表格/记录/字段，不能创建「表格内文档」
  - ✅ 正确做法：创建独立云文档 → 在表格描述中添加链接 → 用户点击访问

### [2026-03-27 16:30] 视图和数据表创建能力边界
- **场景**：用户要求创建「Agent 注册」视图或数据表
- **问题**：尝试调用 API 创建视图和数据表，但都失败了
- **原因**：
  1. **视图管理**：飞书官方 API 支持（`feishu_bitable_app_table_view.*`），但 OpenClaw 工具未实现
  2. **数据表管理**：飞书官方 API 支持（`feishu_bitable_app_table.*`），但 OpenClaw 工具未实现
  3. **认证问题**：尝试直接调用飞书 API，但无法获取正确的 Access Token
- **OpenClaw 已实现的工具**：
  - ✅ `feishu_bitable_create_app` - 创建多维表格应用
  - ✅ `feishu_bitable_create_field` - 创建字段（列）
  - ✅ `feishu_bitable_create_record` - 创建记录（行）
  - ❌ `feishu_bitable_create_table` - 创建数据表（Sheet）- 未实现
  - ❌ `feishu_bitable_create_view` - 创建视图 - 未实现
- **解决方案**：
  1. **创建视图**：用户手动在飞书界面创建（1 分钟）
  2. **创建数据表**：用户手动在飞书界面创建（2 分钟）
  3. **使用现有表**：所有记录在一个表中，通过字段区分类型（推荐）
- **教训**：
  - ✅ 先查阅 `tool-capabilities.md` 确认 OpenClaw 工具是否实现
  - ✅ 官方 API 支持 ≠ OpenClaw 工具支持
  - ✅ 无法创建视图/数据表时，提供替代方案（手动/使用现有表）
  - ✅ 不要尝试绕过 OpenClaw 工具层直接调用 API（token 管理复杂）
