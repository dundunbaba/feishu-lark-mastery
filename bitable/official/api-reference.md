## 消息 API（im/v1/messages）

### 关键参数
- `receive_id_type`: open_id / union_id / user_id / email / chat_id
- `msg_type`: text / post / image / file / audio / media / sticker / interactive / share_chat / share_user / system

### 限制
- 单聊限频: 5 QPS
- 群聊限频: 5 QPS（群内机器人共享）
- 文本消息: 最大 150KB
- 卡片/富文���消息: 最大 30KB
- 图片需先上传（im/v1/image/create），再用 image_key 发送
- 文件需先上传（im/v1/file/create），再用 file_key 发送

### 常见错误码
- 230002: 机器人不在群里
- 230006: 未启用机器人能力
- 230013: 用户不在可用范围 / 已离职
- 230020: 触发频率限制
- 230022: 消息含敏感信息
- 230025: 消息体过大

## 多维表格 API（bitable/v1）

### 新增记录字段格式
- **文本**: 字符串
- **数字**: 数字
- **单选**: 选项值字符串（新选项自动创建）
- **多选**: 字符串数组
- **日期**: 毫秒级时间戳
- **复选框**: true / false
- **人员**: `[{"id": "ou_xxx"}]`（需匹配 user_id_type）
- **超链接**: `{"text": "显示文本", "link": "https://..."}`
- **附件**: `[{"file_token": "xxx"}]`（需先通过 drive/v1/media/upload_all 上传）
- **单向/双向关联**: 记录 ID 数组
- **地理位置**: `"经度,纬度"` 字符串
- **群组**: `[{"id": "oc_xxx"}]`

### 限制
- 单次新增: 最多 500 条
- 总记录: 最多 20,000 条
- **不支持并发写入**（同一张表串行处理）
- 幂等写入: 传 client_token（uuidv4 格式）
- 字段名必须精确匹配（包括空格、特殊字符）

### 常见错误码
- 1254003: app_token 错误
- 1254004: table_id 错误
- 1254045: 字段名不存在（注意精确匹配）
- 1254066: 人员字段格式错误
- 1254103: 记录数超限(20000)
- 1254104: 单次添加超限(500)
- 1254291: 并发写冲突
- 1254607: 数据未就绪（前置操作未完成）

## 文档 API（docx/v1）

### Block 类型（常用）
- 1: 页面 / 2: 文本 / 3-11: 标题1-9
- 12: 无序列表 / 13: 有序列表
- 14: 代码块 / 15: 引用 / 17: 待办
- 18: 多维表格 / 22: 分割线 / 27: 图片
- 31: 表格 / 32: 表格单元格

### 文本样式
- 支持: 加粗/斜体/删除线/下划线/行内代码
- 背景色: 15 种（浅红/浅橙/浅黄/浅绿/浅蓝/浅紫等）
- 字体颜色: 7 种（红/橙/黄/绿/蓝/紫/灰）

### 特殊元素
- @用户: mention_user + user_id
- @文档: mention_doc + token + obj_type
- 链接: text_run.text_element_style.link.url（需 url_encode）
- 日期提醒: reminder（毫秒时间戳）

### 限制
- 频率限制: 5 QPS（单应用）
- 需要文档阅读/编辑权限
- 分页: 最大 page_size=500

## 通用经验

### 权限模式
- `tenant_access_token`: 应用身份（需管理员授权）
- `user_access_token`: 用户身份（OAuth 授权）
- 优先用 user_access_token（权限边界更清晰）

### ID 类型
- open_id: 用户在某应用中的身份（不同应用不同）
- union_id: 用户在同一开发商下的身份（跨应用一致）
- user_id: 用户在租户内的身份（所有应用一致）
- 推荐: 单应用用 open_id，跨应用用 user_id

### 最佳实践
1. 写多维表格前先调用「列出字段」确认字段名精确匹配
2. 批量写入注意串行，不要并发
3. 附件先上传再写入记录
4. 日期字段统一用毫秒时间戳
5. 人员字段注意 user_id_type 匹配
