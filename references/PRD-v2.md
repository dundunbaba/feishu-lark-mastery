## 一、产品概述

### 1.1 产品定位

**Feishu Mastery** = 飞书全能力操作技能 + 知识库引导系统 + 经验共享生态

**目标用户**：
- **主要**：中国国内飞书用户
- **次要**：国际 Lark Suite 用户
- **场景**：OpenClaw 调用飞书套件时的辅助技能

**核心价值**：
- 帮助用户降低使用飞书套件时的配置问题
- 减少调用失败，提高成功率
- 经验共享，减少重复踩坑
- 持续迭代，越用越智能

---

### 1.2 用户画像

| 角色 | 描述 | 需求 | 使用场景 |
|------|------|------|----------|
| **飞书开发者** | 使用飞书开放平台 API 的开发者 | 快速查找 API 文档、避免配置错误 | 开发飞书应用、机器人 |
| **OpenClaw 用户** | 使用 OpenClaw 调用飞书工具的用户 | 减少调用失败、提高成功率 | 日常工作中调用飞书 API |
| **企业管理员** | 倍思科技内部管理员 | 统一飞书 API 使用规范 | 团队内部培训和规范制定 |
| **开源贡献者** | 全球飞书/Lark 生态参与者 | 贡献经验、共同维护 | GitHub 社区协作 |

---

### 1.3 痛点分析

**当前问题**：
1. **API 文档分散** - 飞书开放平台文档分散，查找困难
2. **配置错误频发** - 参数格式、权限配置容易出错
3. **权限问题难排查** - 不知道哪些权限需要申请
4. **需求描述不清** - 用户需求模糊，导致调用失败
5. **无解场景浪费时间** - 飞书 API 不支持的功能，用户反复尝试
6. **经验无法沉淀** - 踩坑经验无法共享，其他人重复踩坑
7. **调用后无反馈** - 用户不知道使用效果，无法感知进步

**影响**：
- 开发效率低（每次调试 1-2 小时）
- 调用失败率高（>30%）
- 用户体验差（反复失败后放弃）
- 知识无法传承（新人重复踩坑）
- 无法持续优化（缺少反馈数据）

---

### 1.4 核心价值

**解决方案**：
1. **知识库引导** - 调用前自动查阅飞书官方文档
2. **渐进披露学习** - 从速查表到官方文档，层层深入
3. **失败触发澄清** - 连续失败 2 次后引导用户精准描述需求
4. **成熟度检查** - 调用前评估，低成熟度强制学习
5. **4 场景诊断** - 快速定位问题原因（未学习/权限/技巧/无解）
6. **经验库建设** - 收集全球用户反馈，持续优化
7. **调用后报告** - 生成使用报告，提升用户体感

**价值主张**：
- **对开发者**：从 2 小时缩短到 10 分钟，降低调试成本
- **对新手**：10 分钟上手，无需老员工指导
- **对生态**：经验共享，减少重复踩坑
- **对产品**：持续迭代，越用越智能

### 2.2 功能详情

#### 功能 1：知识库引导系统

**L1-L4 渐进披露**：

| Level | 什么时候读 | 读什么 | 解决率 |
|-------|-----------|--------|--------|
| **L0** | 每次自动加载 | SKILL.md（操作纪律） | - |
| **L1** | 识别模块 | INDEX.md（路由表） | 定位 |
| **L2** | 常规操作 | 模块/CHEATSHEET.md | 80% |
| **L3** | 不确定/报错 | 模块/official/*.md | 95% |
| **L4** | 本地没答案 | open.feishu.cn 在线搜索 | 99% |

**知识管理规则**：

| 知识类型 | 存放位置 | 更新触发 | 规则 |
|----------|---------|---------|------|
| **官方知识** | `模块/official/` | 定期同步/报错时补充 | 纯官方内容，不掺经验 |
| **经验知识** | `模块/experience/` | 每次操作后 | 标注时间 + 场景 + 结果 |
| **场景方案** | `模块/scenarios/` | 项目完成后 | 标注来源 + 适用条件 |
| **速查表** | `模块/CHEATSHEET.md` | 经验积累到一定量 | 从 official+experience 提炼 |

#### 功能 2：成熟度检查系统

**5 维度评分**：
1. **调用次数** - 历史调用频率
2. **成功率** - 历史成功比例
3. **最近使用** - 最近调用时间
4. **复杂度** - 操作复杂程度
5. **失败处理** - 失败后是否学习

**链路综合评分**：
```
链路成熟度 = Σ(工具成熟度 × 权重) / Σ权重
```

**执行机制**：
- ≥3.0 → ✅ 直接执行
- <3.0 → 🚨 强制学习

| **C: 技巧问题** | 权限正常但调用失败 | 识别需求矛盾，引导澄清 | 显性化矛盾点 |
